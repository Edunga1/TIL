Set - 집합

해시 테이블의 일종

  • 정렬되지 않음
  • Key, Index, Sequence가 없음
  • 중복을 허용하지 않음
  • Fast lookup

어디에 사용될까 곰곰히 생각해보니 제법 제한적일 것이라고 생각되었다.

이유는 해시로 저장하므로 다시 집합에서 꺼내올 일이 없고

중복 허용을 하지 않는 다는 것은 오브젝트간 하나의 그룹에 속해있는지 용도로만 사용하게 될 것이기 떄문이다.
